# 晨報管線維護紀錄

> 晨報／美股／主動ETF 管線在本 repo（`taiwan-flow-live-v2`）。
> 前端 `index.html` 從 GitHub Pages 讀 `data/*.json`；資料由 5 個排程 workflow 產出。

## 2026-07-06 ~ 07-07

### 修掉的三個問題（重要性排序）

**1. Pages 不自動部署（結構性根因，影響最大）**
- 現象：晨報／美股資料有 commit 進 repo，但網站不更新（07-07 早上「晨報未更新」「美股未更新」的真正原因）。
- 根因：前端從 Pages 讀資料，但 GitHub 規定 **bot（GITHUB_TOKEN）的 commit 不觸發 push workflow**，排程資料更新從不重新部署。過去看似會更新，全靠開發期間人工 push 順帶帶上站。
- 修法：5 個資料 workflow（us/morning/aetf/lastweek/baseline）commit 後改用 `gh workflow run pages.yml` 主動 dispatch（workflow_dispatch 不受 bot 限制）。pages.yml 另加：① 部署失敗等 30 秒重試一次（對付 GitHub Pages 後端偶發 `Deployment failed, try again later`）；② checkout 固定 `ref: main`（修掉「push 後即 dispatch，部署綁到前一 commit」的 race）。

**2. 晨報新聞週末缺口**
- 根因：`src/build_morning.py` 的 `overnight_news()` 原本固定 `since = today-1` 且只傳 `start_date`。TaiwanStockNews 是單日查詢，週一早上 `since=週日`，週五收盤後~週六的新聞整段落在視窗外。
- 修法：視窗起點改為「上一個有效交易日（用已抓到的 TAIEX 最近收盤日推得）14:00」，逐日 `start_date=end_date` 迴圈抓到今天再合併；週末／連假自動跳過非交易日、不寫死回溯天數（8 天上限防呆，TAIEX 取不到退回 3 天保底）。既有去重／排論壇／每股≤2／共≤10 邏輯沿用，平日行為不變。**驗收點：下週一早上**。

**3. FinMind 美股資料源延遲（07-07 首次「昨夜有交易」的早晨暴露）**
- 根因：FinMind `USStockPrice` 日線收盤後**逐檔滾動入庫**。07-07 實測：06:46 全無 → 07:36 指數/台積ADR/NVDA 先到 → ~08:25 才全 24 檔齊。瓶頸在資料源。
- 修法：us.yml 加新鮮度重試（日期未推進或 >1/3 檔仍舊 → 等 10 分鐘重跑，最多 12 次≈2 小時）＋**分批上站**（每輪內容有變即 commit＋dispatch，指數/ADR 先到先見，補齊前表格該列標紅「(日期舊)」）。

### 資料時間呈現改版
- 共用 `agoTxt`（<1h→「x分前」、當天→「今天 HH:MM」、更早→「M-D HH:MM」，本地時區）＋ `metaLine`。
- 晨報標「資料日＋產生時間」；美股**分開標行情日與抓取時間**（行情日落後抓取 >4 天 → ⚠資料源延遲）；主動ETF 標「快照日＋抓取時間」，總覽表「揭露日」改「**資料日 d0→d1**」。
- 過舊 ⚠：平日 >26h 未更新、aetf 平日 20:00 後仍非今日快照。
- 後端：lastweek.json 補 `generated_at`。

### 排程最終狀態（台北時間）
| workflow | cron (UTC) | 台北 | 備註 |
|---|---|---|---|
| us | `0 21 * * 0-4` | 05:00 | 重試 12 次、分批上站，目標 08:00 前主力資料到位 |
| morning | `0 22 * * 0-4` | 06:00 | 原 06:20 提早 |
| aetf | `30 10 * * 1-5` | 18:30 | 未動 |
| baseline | `30 12 * * 1-5` | 20:30 | 未動 |
| lastweek | `0 1 * * 1` | 週一 09:00 | 未動 |

- cron 延遲觀察：aetf 首次延遲 3.5h（新 workflow 首跑最慢）；us/morning 約 70 分鐘。
- **「08:00 前全 24 檔美股新資料」不可保證**（取決於 FinMind 入庫，07-07 是 08:25）；主力資料（指數/短評/ADR）07:40 左右可達成。若 FinMind 系統性太慢，下一步考慮換/加更快資料源（如 Yahoo Finance 直抓）。
