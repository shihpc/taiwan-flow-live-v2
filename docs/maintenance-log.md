> 晨報／美股／主動ETF 管線在本 repo（`taiwan-flow-live-v2`）。
> 前端 `index.html` 從 GitHub Pages 讀 `data/*.json`；資料由 5 個排程 workflow 產出。
> Cloudflare Worker 另從 repo raw 讀 baseline.json / lastweek.json 供即時資金流偵測。

## 已修正問題（2026-07-06 ~ 07-07）

### 1. Pages 不自動部署（結構性根因，影響最大）
- **現象**：晨報／美股資料有 commit 進 repo，但網站不更新（07-07 早上「晨報未更新」「美股未更新」的真正原因）。
- **根因**：前端從 Pages 讀資料，但 GitHub 規定 **bot（GITHUB_TOKEN）的 commit 不觸發 push workflow**，排程資料更新從不重新部署。過去看似會更新，全靠開發期間人工 push 順帶帶上站。
- **修法**：5 個資料 workflow commit 後改用 `gh workflow run pages.yml` 主動 dispatch（workflow_dispatch 不受 bot 限制）。pages.yml 另加 ① 部署失敗等 30 秒重試一次（對付 Pages 後端偶發 `Deployment failed, try again later`）；② checkout 固定 `ref: main`（修掉「push 後即 dispatch，部署綁到前一 commit」的 race）。

### 2. 主動加減碼：統一改「股數差法」（演算法核心）
- **驗證結論**：統一（00981A/00403A）**股數逐日確實會變動且可靠**——07-06→07-07 台光電 +111張、聯發科 +94張、京元電子 −1541張、欣興 −436張皆為真實交易。
- **原方法問題（權重差法）**：統一無 units，原以「Δ權重% × AUM」估主動額；但**權重隨股價漲跌變動**，把純價格效應誤判為主動加減碼。07-07 diff 報 00981A 17 訊號、00403A 19，真交易各僅 4~5 筆——約 **3/4 是噪音**。典型：台積電股數兩日沒動（11,960,000），權重卻 10.33→10.77 純因股價，被記成 +529張「加碼」。
- **修法**：改用與其他三家一致的股數差公式（主動Δ = 實際股數 − 前日股數×申贖比 ratio）。統一無 units，`ratio` 以**各持股「今/前股數比」的中位數**推估（多數持股未交易→只隨申贖等比縮放→中位數即申贖比，主動交易為偏離者）。同時免疫申贖與價格效應。
- **效果**：00981A 訊號 17→**5**、00403A 19→**4**，台積電假加碼消失；修後皆為真實動作且有金額（台光電 +10.0億、聯發科兩檔同買 +8.2億、京元電子 −4.9億…）。

### 3. 四家 PCF 端點脆弱性：實證 + 加重試
- **實證**：07-07 一晚三次抓取各有不同端點暫時性掛掉——排程 run 復華 `RemoteDisconnected`、補救重跑換群益 `504 Gateway Timeout`。原程式單次失敗即放棄、該檔掉出快照 → diff 單邊或停舊日。
- **修法**：per-ETF 重試 3 次（退避 5/10 秒）＋全數重試仍失敗則沿用上次快照並標 `stale`。**緩解但非根治**——端點改版仍會壞。

### 4. 晨報新聞週末缺口
- **根因**：`build_morning.py` 的 `overnight_news()` 原固定 `since = today-1` 且只傳 `start_date`。TaiwanStockNews 為單日查詢，週一早上 `since=週日`，週五收盤後~週六的新聞整段落在視窗外。
- **修法**：視窗起點改為「上一有效交易日（TAIEX 最近收盤日）14:00」，逐日 `start_date=end_date` 迴圈抓到今天再合併；週末/連假自動跳過非交易日、不寫死回溯天數（8 天上限防呆，TAIEX 取不到退回 3 天）。既有去重/排論壇/每股≤2/共≤10 沿用，平日行為不變。**驗收點：下週一早上**。

### 5. FinMind 美股資料源延遲
- **根因**：FinMind `USStockPrice` 日線收盤後**逐檔滾動入庫**。07-07 實測：06:46 全無 → 07:36 指數/台積ADR 先到 → ~08:25 才全 24 檔齊。瓶頸在資料源，非排程。
- **修法**：us.yml 加新鮮度重試（日期未推進或 >1/3 檔仍舊 → 等 10 分鐘重跑，最多 12 次≈2 小時）＋**分批上站**（每輪內容有變即 commit＋dispatch，指數/ADR 先到先見；表格對個別舊資料列標紅「(日期舊)」）。

### 6. 資料時間呈現改版
- 共用 `agoTxt`（<1h→「x分前」、當天→「今天 HH:MM」、更早→「M-D HH:MM」，本地時區）＋ `metaLine`。
- 晨報標「資料日＋產生時間」；美股**分開標行情日與抓取時間**（行情日落後抓取 >4 天 → ⚠資料源延遲）；主動ETF 標「快照日＋抓取時間」，總覽表「揭露日」改「**資料日 d0→d1**」。
- 過舊 ⚠：平日 >26h 未更新、aetf 平日 20:00 後仍非今日快照。後端 lastweek.json 補 `generated_at`。
