## 2. 可爬端點詳細證據

### 2a. 沿用既有4家投信端點（只需加 fundId/FundNo，程式改動最小）

**00992A 群益台灣科技創新**（同 `grab_capital`，`fundId` 從 "399" 改成 "500"）
```
POST https://www.capitalfund.com.tw/CFWeb/api/etf/buyback
body: {"fundId":"500","date":null}
```
實測回應（2026-07-13）：`fundName:"群益台灣科技創新主動式ETF基金"`，`stocks[0]={stocNo:"2330",stocName:"台積電",share:1406000,weight:8.118}`，`totUnit:2346955000`，欄位與既有 00982A 完全一致，直接複用 `grab_capital("500")`。
（順帶確認 fundId=501 是群益標普500ETF、502 是群益美國增長00997A——皆非台股標的，跳過。）

**00985A / 00999A 野村系**（同 `grab_nomura`，`FundNo` 直接用 ETF 代號本身，不需另查ID）
```
POST https://www.nomurafunds.com.tw/API/ETFAPI/api/Fund/GetFundTradeInfoDate  {"FundNo":"00985A","Type":2}
POST https://www.nomurafunds.com.tw/API/ETFAPI/api/Fund/GetFundTradeInfo      {"FundNo":"00985A","Type":2,"Date":"<LatestDate>"}
```
實測：00985A `CFullName:"野村臺灣增強50主動式ETF"`（簡稱"主動野村台灣50"）；00999A `CFullName:"野村臺灣策略高息主動式ETF"`。兩者皆回傳 `CAnceTotalIssues`（單位數）、`CAnceTotalAv`（AUM）、`Stocks[]`，直接複用 `grab_nomura("00985A")` / `grab_nomura("00999A")` 即可，程式碼一行不用改，只需在 ETFS 清單加兩筆。

### 2b. 新投信端點（需新寫 grab 函式）

**00994A 第一金台股優** — 正式 WebAPI（ASP.NET）
```
POST https://www.fsitc.com.tw/WebAPI.aspx/Get_hd
Content-Type: application/json; charset=utf-8
body: {"pStrFundID":"182","pStrDate":""}
```
回應是雙重編碼 JSON（`{"d":"[...]"}`，`d` 內是字串化的 JSON array，需再 `json.loads` 一次）。
實測片段：
```json
{"fundid":"182","sdate":"2026-07-09","group":"1","A":"2330","B":"台積電","C":"16.48","D":"345,999","E":""}
{"fundid":"182","sdate":"2026-07-09","group":"1","A":"2383","B":"台光電","C":"7.32","D":"70,000","E":""}
```
欄位對應：A=股票代號、B=股票名稱、C=持股權重(%)、D=股數、E=備註（空）、group="1"為股票（其他group值可能是期貨/現金，未逐一確認）。`pStrFundID`固定182是00994A的內部ID（從fsitc.com.tw官網頁面 `var pStrFundID = '182';` 讀出）。全量持股、無反爬阻擋。

**00405A 富邦台灣龍耀** — 伺服器端渲染HTML表格
```
GET https://websys.fsit.com.tw/FubonETF/Fund/Assets.aspx?stkId=00405A
```
純HTML `<table>`，無需JS執行、無API。欄位：代號/名稱/股數/市值/權重（`<td class="tac">`），實測見台積電列 `2330｜台積電｜239,000｜577,185,000｜2.2356`，全表約50檔。用正則或BeautifulSoup解析`<tr><td>`即可，不需登入、無防爬。

**00996A 兆豐台灣豐收** — 伺服器端渲染HTML（div排版非table標籤）
```
GET https://www.megafunds.com.tw/MEGA/etf/etf_product.aspx?id=23
```
資料在 `<div class="fund-content">` 裡逐欄位排列（代號/名稱/股數/權重四個div為一組），實測：`2330｜台積電｜179,000｜8.92 %`。無API，直接正則抓 `class="fund-content"` 系列div即可。

**00987A（及00986A，但為全球標的排除）台新** — 伺服器端渲染HTML table
```
GET https://www.tsit.com.tw/ETF/Home/ETFSeriesDetail/00987A
```
標準 `<table class="table table-striped">`，欄位：代號(含"TT"字尾)/名稱/股數/持股權重。實測：`2330 TT｜台積電｜90,000｜7.9760%`。無需登入。程式需把代號字尾" TT"去掉。

## 3. 未成功的投信（列原因，供後續攻堅參考）

- **國泰投信 00400A**：cathaysite.com.tw 頁面用瀏覽器UA可正常取得（curl 200，WebFetch工具本身403），
  但1MB靜態HTML中找不到持股table或`__NEXT_DATA__`/API路徑字串，懷疑是前端XHR動態載入、
  API host可能是 `cwapi.cathaysite.com.tw` 但確切path未找到。需要用瀏覽器開發者工具實際錄製network再確認。
- **摩根投信 00401A**、**聯博投信 00404A**：官網為行銷頁性質，00404A頁面雖含"2330"等股票代號，
  但只是行銷用「舉例個股」表格（3-5檔示意），非每日PCF全量持股，未找到正式揭露頁/API。
- **中國信託投信（00406A/00983A/00995A）**：官網 `ctbcinvestments.com.tw` 是 Vue/React SPA
  （`<div id="app">`空殼），且被 Incapsula WAF 保護（跟群益的CFWeb API擋非法fundId時回應的Incapsula
  是同一套機制），下載JS bundle（79萬字元）搜尋未見明顯`/api/`路徑字串（可能是動態組字串或另有子網域）。
- **凱基投信 00407A**：抓到的頁面（Upload/Activity活動頁）非正式基金資料頁，需另找
  `kgifund.com.tw` 上正式ETF持股揭露頁面，本次未查到。
- **安聯投信 00984A/00993A**：`etf.allianzgi.com.tw` 首頁僅9KB，同樣是SPA空殼，未深入。

## 4. 建議優先順序

1. **00992A、00985A、00999A**（複用既有4家端點，改動最小，程式風險最低，建議優先上）——
   加上這3檔後，統一系仍2檔、群益系變2檔、野村系變3檔、復華系仍1檔，共8檔。
2. **00994A 第一金**、**00405A 富邦**、**00996A 兆豐**、**00987A 台新**——
   4個新投信、新端點但技術簡單（純HTML表格或已知格式JSON API，無反爬），建議次優先，共可再加4檔。
3. 以上7檔全部到位後共12檔，涵蓋7家投信；規模上00992A（群益台灣科技創新，419億）、
   00405A（富邦台灣龍耀，242億）屬於較大型主動ETF，資金流訊號意義較高，可優先做。
4. 國泰、摩根、聯博、中國信託、凱基、安聯共6家投信仍未破解，建議之後有餘裕時用瀏覽器
   開發者工具（Network tab）實際操作官網頁面，找真正的XHR/fetch API，靜態curl法已到極限。
